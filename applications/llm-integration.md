# LLM-assisted Workflow

This document describes application concepts for using Large Language Models (LLMs) together with E2R datasets.

This document is informational and is not part of the E2R Core Specification.

## Design Philosophy

The application does not require direct integration with any specific LLM service.

Instead, it supports a human-centered workflow:

1. The application exports a subset of an E2R dataset together with instructions.
2. The user copies the generated text.
3. The user sends the text to any LLM of their choice.
4. The LLM returns E2R-compatible JSON.
5. The application validates and merges the returned data.

This workflow avoids vendor lock-in and allows users to choose any compatible LLM.

## Possible Assistance

### Entity Creation

- Extract entities from text.
- Suggest entity names.
- Suggest aliases.
- Generate descriptions.

### Event Creation

- Extract events.
- Infer chronological order.
- Generate event descriptions.

### Relation Creation

- Suggest missing relations.
- Infer relationship candidates.
- Detect isolated objects.

### Dataset Completion

- Detect duplicate entities.
- Suggest object merges.
- Detect incomplete information.
- Suggest missing participants.

### Validation Assistance

- Explain validation errors.
- Explain merge conflicts.
- Suggest corrections.

### Analysis

- Summarize datasets.
- Explain relationship networks.
- Identify central entities.
- Detect possible inconsistencies.

### Creative Assistance

- Generate timelines.
- Organize fictional worlds.
- Review story consistency.
- Suggest missing events.

## Prompt Export

Applications may generate plain text prompts containing:

- Task description
- Editing instructions
- Relevant subset of an E2R dataset
- Output format requirements

The generated prompt is intended for manual submission to an LLM.

## JSON Import

Applications may import JSON generated by an LLM after validation.

Imported data should be validated before merging into the dataset.

## Non-goals

This document does not require:

- API integration
- Online services
- Specific LLM providers
- Cloud connectivity

All interaction may be performed manually by the user.
